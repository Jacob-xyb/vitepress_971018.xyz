# Git Commit Message 规范

当生成 Git commit message 时，请遵循以下规则：

1. **使用中文**：所有 commit message 必须使用中文
2. **格式规范**：
   - feat: 新功能
   - fix: 修复问题
   - docs: 文档更新
   - style: 代码格式调整
   - refactor: 重构代码
   - test: 测试相关
   - chore: 构建/工具相关

3. **示例**：
   - `feat: 添加用户登录功能`
   - `fix: 修复导航栏显示问题`
   - `docs: 更新 README 文档`

4. **描述要求**：
   - 简洁明了，不超过 50 字
   - 使用动词开头
   - 说明做了什么，而不是为什么
